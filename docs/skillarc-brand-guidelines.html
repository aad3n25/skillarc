04 — Motion
Animation System
Smooth, purposeful animation that reinforces actions without distracting from content.
Float (Background Blobs)
Slow, organic movement on decorative background shapes. 12s duration, infinite loop.
animation: float 12s ease-in-out infinite;
@keyframes float {
  0%,100% { transform: translateY(0); }
  50% { transform: translateY(-20px); }
}
Hover Lift (Cards)
Subtle Y-axis lift + background brightening on card hover. Immediate and responsive.
transition: all 0.25s ease;
hover:bg-white/20
hover:border-white/40
hover:-translate-y-0.5
Button Press
Gradient deepens + subtle lift on hover. Communicates interactivity immediately.
transition: all 0.2s;
hover:from-purple-700 hover:to-blue-700
hover:-translate-y-px
hover:shadow-purple-500/40
Modal Fade-In
Overlay and modal panels fade and scale in. Creates smooth context switching.
@keyframes fadeIn {
  from { opacity:0; transform: scale(0.97); }
  to { opacity:1; transform: scale(1); }
}
animation: fadeIn 0.2s ease-out;
Timing Principles
Use 0.2s for micro-interactions (buttons, badges, inputs). Use 0.25s for card-level transitions. Use 0.3–0.5s for modals, drawers, and page elements. Background floats use 10–14s for ambient, non-distracting motion. Always use ease or ease-in-out — never linear.
05 — Principles
Design Principles
The five rules that govern every design decision across the platform.
01
Glassmorphic Consistency
Every surface — cards, modals, inputs, nav — uses the same glass aesthetic: bg-white/10, backdrop-blur, white border. Never use flat or solid-filled cards.
02
Gradient Identity
Purple-to-blue is the brand signature. It appears on primary buttons, active states, decorative blobs, and text gradients. Use it intentionally — never overuse.
03
Clear Visual Hierarchy
Size, colour, and spacing create flow. Large bold headings → subheadings → body text → metadata. Users should never need to hunt for what to focus on.
04
Smooth Interactions
Every interactive element has a hover state. Everything transitions. No instant jumps. Animation duration scales with the size of the change — micro = fast, macro = slower.
05
Mobile-First Responsive
Base styles target small screens (px-4, compact spacing). Use sm: and md: prefixes to enhance for wider viewports. Never design desktop-first and scale down.
06
High Contrast Text
White text on dark backgrounds. Dark gray on light cards. Never use low-opacity text for anything functional — reserve rgba(255,255,255,0.5) for supporting/meta text only.
Responsive Breakpoints
/* Mobile first (default) */ padding: px-4 py-20; font-size: text-2xl; max-width: w-full; /* Tablet (sm: 640px+) */ padding: sm:px-8; font-size: sm:text-4xl; /* Desktop container */ max-width: max-w-3xl mx-auto; /* assessment flow */ max-width: max-w-6xl mx-auto; /* dashboard / wide layouts */
